## ① SPEF 四阶段与边界　〔手册 §2.1〕

| 环节 | 做什么 | **不做什么** |
|---|---|---|
| **Scan** | 读准真实状态 | 只读现实，不出方案 |
| **Plan** | 基于扫描出方案 | 只出意图，不动手 |
| **Execute** | 实施方案 | 只忠实施加，不夹带判断 |
| **Feedback** | 验证、闭环 | **必须有人** |

- 理想配置：S/P/E 交给 Agent，**F 留给人**。
- 核心准则：正确性不靠"某步做对"，靠"**这个环能否稳定闭合**"。
- **只闭那个只有人能闭的环**，其余交给编译器/测试。
- S/P 交织处 = 双高节点（高耦合 × 高不可观）= 最该押注的地方。

## ② 五秒赌注模板　〔手册 §2.2〕

> 在押一个高风险点"应该没问题"的当下，写：
>
> **「我赌此处无误，因为 ___」**
>
> 写成赖不掉的硬陈述 → **看结果前** 带时间戳落盘（git 提交）。

**自信度三档**：　🟢 高　/　🟡 中　/　🔴 低　（与赌注一起记）

**两条通道（承重不同，别混）**：

- **通道一 预测—结算**：只靠"存在 + 可证伪 + 现实结算"。confabulation 腐蚀不到它 → **承重主力**。
- **通道二 写"因为"**：靠 self-explanation 暴露裂缝。confabulation 严重时失效 → 锦上添花。

⚠️ 别用"两个一致的因为"去抓 confabulation（会奖励最会圆话的人）。两个"因为"的用途是**扩大可证伪面、提升分辨力**。

## ③ 押注判据 L 三区制　〔手册 §2.3〕

$$L = \dfrac{R_L\ (\text{覆盖的真实风险占比})}{\phi\ (\text{主观押注覆盖率})}$$

| 区制 | 含义 | 动作 |
|---|---|---|
| **L > 1** | 押对，带宽压在高风险处 | 保持 |
| **L ≈ 1** | 橡皮图章 / 昂贵均摊 | 停止均摊，重新聚焦 |
| **L < 1** | **显著性陷阱**（专看不出事处） | 警报：勤奋正在加害 |

- 按**区制**读，不按数值线性比；**周期聚合**后读，非单次。
- 深浅分流：浅问题重押直觉（错了不亏）；深问题早转扫描。
- **最危险信号**：扫得远超预期、概率却不升 → bug 在边界外 → 跳出去重扫边界假设。

## ④ 红线与终止条件　〔手册 §2.5〕

**开工前冻结三项，事后一律不得回调**：

1. **故障规则 R** —— 钉死"算不算一次故障"，禁止看着数据反向调 R。
2. **Kill-switch** —— 预先承诺停做条件（终止权不交给未来的你）。
3. **诚实崩溃触发** —— 一旦回填时间戳 / 事后调 R → 判 E5 已塌 → **立即终止**。

**预承诺终止线**：到第 X 月，$N(t)$ 的 $b$ 仍低于阈值 **且** 找不到一个"当下的我会漏、仅靠记账捞回"的故障 → 终止。

**动机别从校准曲线榨**：命中率"维持"是右删失、无信息量，对它正强化 = 重建自欺。
**动机挂当下吞吐量**：本月台账定位 X 个故障；本周 Y 个赌注揭晓、Z 个当场改写了我的判断。
