§1.1 · Starting point
Why is optimizing Agent workflows so difficult?
Lack of a standard of acceptance independent of the operator's subjective judgment
↳ Problem statement: the most dangerous errors stay green the whole time, never trigger any automatic alarm
▼
causes
§1.1 · Amplification mechanism
Three blind spots compound
Three independent cognitive failure mechanisms running simultaneously, mutually amplifying
① Salience ≠ risk
Attention rushes to "looks suspicious" places; deadliest failures stay green (salience bias / WYSIATI)
② Inspected: no echo
"Checked, found nothing" and "actually fine" are subjectively indistinguishable; operator never gets "you missed it"
③ Diligence backfires
First two compound: exhaust bandwidth on low-risk conspicuous places, miss high-risk hidden places, get false confidence

constitute jointly
§1.2 · Book's pivot KEY
The core dilemma: self-reference
Measurer (your judgment) = thing measured (you) → any present self-assessment is structurally unreliable
↳ Pivot of all later argument. Consequence: termination conditions also pass through this gate; must be pre-committed (→ §2.5 kill-switch)

§1.5 · Line B-2
Formula L: wager quality metric
Not "did you hit"; rather: did your bandwidth allocation truly concentrate on high-risk
↳ Formalizes "how to allocate finite bandwidth"; uses ratio not difference
L = R_L / φ R_L = true risk proportion covered φ = subjective wager coverage estimate L > 1 wagered correctly (risk outpaces coverage) L ≈ 1 rubber stamp (costly averaging) L < 1 salience trap L readable only by regime, not linear magnitude

§1.5 · Line B-3
L<1 is an attractor
High-risk nodes precisely have low salience appearance — formal consequence of salience bias
↳ "Harder work" cannot solve it; L is fundamentally a resolution metric

§2.2 · Metacognitive scaffold  KEY
Instantaneous prediction wager
"I predict this is correct because___" (timestamp-lock before results)
↳ Operationalization of three-line requirements; value comes from two independent channels with different load-capacity and corruption resistance
Two independent value channels — different load-capacity, different corruption resistance
Channel One · Brier / Tetlock component
Prediction—settlement
Effectiveness depends only on wager existence and falsifiability, not on "because ___" being true cause
↳ Regardless of confabulation severity, this channel intact ✓; reality settles; confabulation cannot corrupt
Channel Two · self-explanation
Writing "because" exposes cracks
Forced externalization of mental model before results; requires stated reason genuinely linked to true model
↳ Confabulation's precise attack surface ✗; real value is expanding falsifiability, not resisting confabulation

§2 + §2.5 · Operations layer
§2.1
SPEF four-stage
S→P→E→F functional division, not temporal; F requires human participation
↳ Makes work decomposable, error-locatable; S/P junction is dual-high node (coupling + unobservability)
§2.4
Dead/live variable separation
Dead: timestamps, commits (zero judgment, auto-logged); Live: wagers+confidence (bears E5); lock three items before start
↳ Divorces "is this an error" judgment from present self-assessment; wagers must timestamp before results
§2.4
Degeneration-prevention mirror
Censoring awareness, one-way use, shows only "no degradation," rarely "progress"
↳ Needs resolution second-axis (L is entry); deduct difficulty drift; never positive-reinforce uninformative signal
§2.5
Kill-switch
Pre-commit, lock before start, execute mechanically at time, no present re-negotiation
↳ Direct consequence of self-reference conclusion: termination power does not go to present-tense you; honest method contains its own termination
